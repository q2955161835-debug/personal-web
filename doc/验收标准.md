## 测试环境
- 操作系统：Windows 11，本地 PowerShell。
- Node.js：v22.17.1，npm 10.9.2。
- 浏览器：Chrome / Edge / Firefox / Safari 最新版（需支持 WebGL2）。
- 工作区：`d:\0文件夹\个人站`。

## 验收项 A：项目骨架与本地启动
- 验收步骤：
  1. 运行 `npm install` 安装依赖。
  2. 运行 `npm run dev` 启动开发服务器。
  3. 用 Edge 或 Chrome 打开 `http://127.0.0.1:5173/`。
  4. 等待 1 秒后 loader 隐藏，看到星空背景。
  5. 运行 `npm run typecheck` 确认无类型错误。
- 预期结果：
  - `node_modules/` 安装成功，无致命错误。
  - 开发服务器在 127.0.0.1:5173 启动。
  - 页面加载后 loader（橙色脉冲圆）在 1 秒内淡出。
  - 全屏黑色背景上出现三层星空（远景密集小白点 + 中景彩色星点 + 近景闪烁亮星）。
  - 鼠标移动时相机有轻微视差。
  - 顶部导航显示 "FAN JUNJIE · 范俊杰" 和 GitHub / Contact 链接。
  - 底部显示滚动提示 "SCROLL TO EXPLORE" 和 HUD 信息。
  - `npm run typecheck` 退出码 0。
- 结果记录：
  - 2026-07-05：通过。`npm install` 成功；`npm run dev` 启动成功；`npm run typecheck` 退出码 0；`npm run build` 构建成功（three 504kB / index 20.5kB）。视觉验收由用户在浏览器确认。

## 验收项 J：性能优化与验收（阶段10）
- 验收步骤：
  1. 在 Chrome DevTools Performance 面板录制 5 秒滚动操作，检查 FPS 是否稳定在 60 附近。
  2. 检查任务管理器 GPU 进程内存占用是否合理（< 500MB）。
  3. 切换到其他浏览器标签页，等待 5 秒后切回，检查场景是否正常恢复渲染（无卡死）。
  4. 在系统设置中启用"减少动态效果"（Windows: 设置 → 辅助功能 → 视觉效果 → 关闭动画），刷新页面，检查动画速度是否降低。
  5. 在 Chrome DevTools 中执行 `performance.mark` + `performance.measure` 检查单帧渲染耗时 < 16ms。
  6. 检查 `npm run build` 产物大小：three chunk < 500kB / gzip < 130kB。
  7. 跨浏览器测试：Chrome / Edge / Firefox / Safari 最新版均能正常加载与渲染。
- 预期结果：
  - 滚动时 FPS 稳定 60（允许偶发 55+）。
  - GPU 内存 < 500MB。
  - 标签页隐藏时暂停渲染，恢复时正常。
  - 启用 reduced motion 后动画速度降低（animScale=0.3）。
  - 单帧渲染 < 16ms。
  - three chunk 489kB / gzip 124kB。
  - 4 大主流浏览器均正常。
- 结果记录：
  - 2026-07-05：自动验证通过。`npm run build` 产物 three 489kB / gzip 124kB，index 132kB / gzip 43kB。性能与跨浏览器项需用户人工确认。
