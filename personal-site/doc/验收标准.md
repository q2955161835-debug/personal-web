# 验收标准

## 测试环境
- 本地项目目录：`D:\0文件夹\个人站\personal-site`
- 开发服务：`npm run dev -- -p 3000`
- 验证浏览器：应用内 Browser 或等效 Playwright 浏览器
- 必跑命令：`npm run lint`、`npm run build`
- 视觉验收视口：桌面 1440x900 附近、移动 390x844 附近

## 全局浏览器纵览验收

### 验收步骤
1. 打开 `http://localhost:3000`。
2. 从 Hero 首屏开始，慢速滚动经过 About、Projects、Data Analysis、Experience、Contact 到页面底部。
3. 从 Contact 向上滚回 Projects、About 和 Hero。
4. 在滚动期间截图或观察关键状态，并检查浏览器控制台新增错误。

### 预期结果
- Hero 粒子在首屏正常显示，从 Projects 回滚到上方页面后不消失。
- About 内容与全局 Canvas 不互相遮挡，页面上下没有大段失衡空白。
- Projects 段进入后使用独立 DNA 视觉，Hero 粒子淡出，不再干扰 DNA。
- 滚动过渡连续自然，项目切换节奏跟随鼠标滚轮速度，不出现离散跳变过快。
- Data Analysis、Experience、Contact 段上下衔接紧凑，不出现大段空白或内容漂浮失衡。
- Canvas 始终非空，无主要视觉对象飞出视口或遮挡正文。

### 结果记录
- 2026-07-05：通过。`npm run lint`、`npm run build` 均通过；Edge Chromium + Playwright 完成桌面 1440x900 与移动 390x844 视觉验收，证据位于 `try/visual-qa-20260705-215203/`。应用内 Browser 在读取页面状态时超时重置，已记录为工具异常并改用等效浏览器自动化验收。
- 2026-07-05：本轮通过。`npm run lint`、`npm run build` 均通过；Edge Chromium + Playwright 完成 Hero→About→Projects→Data Analysis→Experience→Contact→回滚 About 的全局纵览，最终证据位于 `try/visual-qa-20260705-222843/`，`result.json` 中所有 pass 项为 true，控制台运行错误为空。

## Projects DNA 项目段验收

### 验收步骤
1. 滚动到 Projects 段。
2. 慢速滚动观察 6 个项目的切换。
3. 点击右侧项目序号跳转到首尾与中间项目。
4. 点击 DNA 键标签或中心项目按钮打开详情。
5. 观察项目标签、左侧状态栏、DNA 主体、背景粒子关系。

### 预期结果
- 项目名称不重叠，固定展示在 DNA 对应附近，不堆在底部。
- 左侧状态栏显示当前项目序号、名称与副标题，文字完整可读。
- 右侧序号导航可点击，激活态与当前项目一致。
- DNA 键标签跟随同一螺旋变换，位置固定在对应键附近，点击标签可触发最大放大与详情打开。
- 相机角度保持稳定，滚动驱动 DNA 自身旋转、向下移动、粒子聚合与项目键缩放。
- 鼠标划过 DNA 粒子时粒子被打散，鼠标移开后回弹复原。
- DNA 主体清晰，后景粒子数量克制，不干扰项目浏览。
- 从 Projects 切回上方页面后，首页粒子和相机状态恢复正常。

### 结果记录
- 2026-07-05：通过。桌面截图 `02-desktop-projects-flow.png` 显示 Hero 粒子已与 DNA 段分离、后景粒子克制、项目标签固定在 DNA 附近；移动截图 `07-mobile-projects-flow.png` 显示移动端仅保留左侧标题与序号，项目名由 DNA 标签承载，无文字重叠。滚轮样本从项目 1 渐进到项目 2、项目 3，未出现一次滚动直接跳到末尾。
- 2026-07-05：本轮通过。最终截图 `03-desktop-dna-gather.png`、`05-desktop-dna-mid-active-key.png`、`15-mobile-dna-projects.png` 显示粒子由项目入口聚合到 DNA、相机角度稳定、键标签跟随 DNA、移动端中心项目按钮不与左侧状态栏重叠；`06-desktop-dna-pointer-scatter.png` 与 `07-desktop-dna-recovered.png` 覆盖鼠标打散与复原。

## 项目详情验收

### 验收步骤
1. 在 Projects 段点击当前项目标签，打开详情层。
2. 分别验证至少一个 `flow` 项目和一个 `chart` 项目。

4. 检查详情层按钮、链接、遮罩、滚动与关闭行为。

### 预期结果
- 详情层不是纯黑空白背景，而是显示与项目类型匹配的主题视觉。
- 数据分析/性能诊断项目展示主线图视觉，量化交易项目展示 K 线图视觉。
- 视觉识别、波形、网格、节点类项目展示对应的图形语义。
- 详情内容在桌面和移动视口内可读，不与背景图形、按钮或遮罩重叠。
- 打开、切换、关闭过程无新增控制台错误。

### 结果记录
- 2026-07-05：通过。桌面详情截图 `03-desktop-flow-detail.png` 展示视觉识别路径图，`05-desktop-chart-detail.png` 展示量化交易 K 线图；移动详情截图 `08-mobile-flow-detail.png` 可读、可关闭、无横向溢出。运行时错误为空，仅有 Three.js 依赖层 `THREE.Clock` 弃用警告。
- 2026-07-05：本轮通过。最终截图 `08-desktop-project-detail.png`、`16-mobile-project-detail.png` 显示项目详情有对应主题背景图形，非纯黑空白；桌面与移动详情均可读、可关闭，`result.json` 中 `projectDetailDesktop=true`、`projectDetailMobile=true`。
